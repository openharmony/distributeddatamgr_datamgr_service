# DFX 规范

> 本文档为 Agent 知识路由目标，由 AGENTS.md 的 task-based / path-based / vocabulary-based routing 触发阅读。

## 1. 适用范围

本文档适用于以下场景：
- 修改日志（HiLog / ZLOG）输出
- 修改故障诊断逻辑
- 修改事件上报或统计信息
- 修改错误码定义

## 2. DFX 规则

### 2.1 日志规范

- 关键业务状态变更必须记录日志（云开关变更、同步开始/结束、冲突发生、订阅状态变更）。
- 日志级别使用规范：
  - DEBUG：开发调试信息，发布版本不输出
  - INFO：关键业务流程节点
  - WARN：可恢复的异常情况
  - ERROR：影响功能的错误
  - FATAL：导致服务不可用的严重错误
- 不得为通过测试删除日志、事件、错误码或诊断信息。

### 2.2 故障归因

- 同步失败必须记录错误码、失败阶段、关联的 store/bundle/user 信息。
- 网络中断导致的同步中断必须记录，并触发补偿同步。
- 重试耗尽必须记录最终失败原因。

### 2.3 统计与上报

- CloudServiceImpl 支持查询统计信息（QueryStatistics）和最近同步信息（QueryLastSyncInfo）。
- 变更统计字段或统计口径时，需确认不影响已有查询方的使用。

## 3. 检查方法

- 确认新增/修改的日志使用了正确的日志级别。
- 确认关键业务路径有日志覆盖。
- 确认错误码变更不会导致已有监控/告警失效。
